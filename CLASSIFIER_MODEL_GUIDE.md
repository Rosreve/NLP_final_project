# Guide: Adding a New Classifier Model

This guide explains how to add a new claim classifier to the current pipeline while keeping the same object-oriented style as the existing code.

The current pipeline is:

1. Load claims and evidence from `data/`.
2. Retrieve evidence for each dev claim with `CombinedRetreiver`.
3. Optionally rerank retrieved evidence with `DistilBertReranker`.
4. Train a classifier on `train-claims.json`.
5. Predict labels for dev claims using the retrieved evidence file.
6. Write predictions to `output/predictions.json`.
7. Evaluate with `eval.py`.

## Important Files

| File | Purpose |
| --- | --- |
| `run.py` | Main pipeline entry point: retrieval, optional reranking, classification, output paths |
| `src/classifier.py` | Current classifier implementation: `TfidfClaimClassifier` |
| `util.py` | Loads training claims, dev claims, and evidence |
| `src/config.py` | Central path and output filename config |
| `src/retreiver/retreiver.py` | Evidence retrievers used before classification |
| `src/retreiver/reranker.py` | Optional DistilBERT cross-encoder reranker |
| `eval.py` | Scores claim labels and retrieved evidence |

Note: this codebase consistently uses the package/folder spelling `retreiver`, so keep that spelling in imports.

## Data Sources

All main input data is loaded through `util.load_data()`(or you can create your own method for loading the data source):

```python
train_claims, dev_claims, evidence = load_data()
```

The paths are defined in `src/config.py`:

| Data file | Config variable | Used for |
| --- | --- | --- |
| `data/train-claims.json` | `TRAIN_CLAIMS_PATH` | Training the classifier. Each claim has `claim_text`, `claim_label`, and gold `evidences`. |
| `data/dev-claims.json` | `DEV_CLAIMS_PATH` | Ground-truth dev labels and evidence for evaluation. |
| `data/evidence.json` | `EVIDENCE_PATH` | Evidence ID to evidence text mapping. |

Claim examples have this shape:

```json
{
  "claim-1266": {
    "claim_text": "This means that the world is now 1C warmer than it was in pre-industrial times",
    "claim_label": "SUPPORTS",
    "evidences": ["evidence-315434", "evidence-198055"]
  }
}
```

The final classifier does not read raw `dev-claims.json` directly for prediction. It reads the retrieval output generated by `run.py`:

```text
output/retreived_data/<retrieval_filename>.json
```

In the current `run.py`, the filename is based on `COMBINED_RETRIEVAL_FILENAME` and whether reranking is enabled. With the default arguments, the classifier receives:

```text
output/retreived_data/dev-claims-combined-retreival_reranked.json
```

Each retrieved dev claim should have:

```json
{
  "claim_text": "...",
  "claim_label": "NOT_ENOUGH_INFO",
  "evidences": ["evidence-id-1", "evidence-id-2"]
}
```

The `claim_label` in retrieved dev data is only a temporary dummy label before prediction.

## Environment And Packages

Use Python 3.10 or newer. The code uses modern type hints such as `str | None`.

Create and activate a virtual environment from the repo root:

```bash
python -m venv .venv
source .venv/bin/activate
```

Install the packages used by the current pipeline:

```bash
pip install numpy scikit-learn tqdm matplotlib nltk sentence-transformers torch transformers accelerate joblib
```

If `BM25Retreiver` raises an NLTK WordNet error, download WordNet once:

```bash

```

The first run may download model weights for:

- `sentence-transformers/all-MiniLM-L6-v2`
- `distilbert-base-uncased`, if reranking is enabled

The pipeline also writes caches under `output/temp_cache/`, including embeddings and combined retriever pickles.

## Current Classifier Style

The existing classifier is `TfidfClaimClassifier` in `src/classifier.py`.

It follows this object-oriented pattern:

1. `__init__` loads data, loads retrieval results, and initializes the model.
2. `_build_classification_data` converts one claim into model input text and label.
3. `_prepare_training_data` builds training `texts` and `labels`.
4. `_prepare_dev_data` builds dev `texts` and `labels` if needed.
5. `train_model` fits the model.
6. `predict_one` predicts one claim label from claim text and evidence IDs.
7. `evaluate` writes predictions to `output/predictions.json`.
8. `_load_retrieval_results` reads retrieval output from `output/retreived_data/`.

For a new classifier, keep the same public methods:

```python
classifier = NewClaimClassifier(retrieval_filename=file_name)
classifier.train_model()
classifier.evaluate()
```

This lets `run.py` swap classifiers with minimal changes.

## How To Add A New Classifier

### 1. Add A New Class In `src/classifier.py`

Add your classifier below the existing `TfidfClaimClassifier`, or put it in a new file only if it becomes large.

Use a class name that describes the model, for example:

```python
class TransformerClaimClassifier:
    ...
```

Keep the constructor shape compatible with the existing pipeline:

```python
def __init__(self, retrieval_filename: str = "dev-claims-combined-retreival_reranked.json"):
    self.train_claims, _, self.evidence = load_data()
    self.dev_claims = self._load_retrieval_results(retrieval_filename)
    self.model = ...
```

### 2. Reuse The Same Input Format

The current classifier combines claim text and retrieved evidence text:

```python
combined_text = claim_text + " [SEP] " + " ".join(evidence_texts)
```

Use this same format unless your model needs token pairs. This keeps experiments comparable.

For transformer models, you can pass the claim and evidence separately to the tokenizer, but still build evidence texts from IDs the same way:

```python
evidence_texts = [
    self.evidence[eid]
    for eid in evidence_ids
    if eid in self.evidence
]
evidence_text = " ".join(evidence_texts)
```

### 3. Implement `train_model`

Training data comes from `train-claims.json`. For each training claim:

- Input: `claim_text` plus gold evidence texts from `evidences`.
- Target: `claim_label`.

The valid labels are:

- `SUPPORTS`
- `REFUTES`
- `NOT_ENOUGH_INFO`
- `DISPUTED`

For scikit-learn models, `train_model` should call:

```python
self.model.fit(train_texts, train_labels)
```

For neural models, `train_model` should:

1. Prepare texts and labels.
2. Map labels to IDs.
3. Tokenize examples.
4. Train the model.
5. Store anything needed for prediction on `self`, such as `label_to_id`, `id_to_label`, tokenizer, and device.

### 4. Implement `predict_one`

`predict_one` must accept the same arguments:

```python
def predict_one(self, claim_text, evidence_ids):
    ...
```

It must return one label string, not a label ID:

```python
return "SUPPORTS"
```

This method is called by `evaluate`.

### 5. Keep `evaluate` Output Compatible

`eval.py` expects `output/predictions.json` to look like this:

```json
{
  "claim-123": {
    "claim_label": "SUPPORTS",
    "evidences": ["evidence-1", "evidence-2"]
  }
}
```

The existing `evaluate` method already writes this format. You can reuse the same method in your new class.

## Minimal Template

Use this as a starting point for a new classifier:

```python
import json

from tqdm import tqdm
from util import load_data
from config import OUTPUT_PATH


class NewClaimClassifier:
    def __init__(self, retrieval_filename: str = "dev-claims-combined-retreival_reranked.json"):
        self.train_claims, _, self.evidence = load_data()
        self.dev_claims = self._load_retrieval_results(retrieval_filename)
        self.model = None

    def _build_classification_data(self, claim):
        claim_text: str = claim.get("claim_text", "")
        claim_label: str = claim.get("claim_label", None)

        evidence_texts = [
            self.evidence[evidence_id]
            for evidence_id in claim.get("evidences", [])
            if evidence_id in self.evidence
        ]

        combined_text: str = claim_text + " [SEP] " + " ".join(evidence_texts)
        return combined_text, claim_label

    def _prepare_training_data(self):
        texts, labels = [], []

        for claim_data in tqdm(self.train_claims.values(), desc="Preparing training data"):
            text, label = self._build_classification_data(claim_data)
            if label is not None:
                texts.append(text)
                labels.append(label)

        return texts, labels

    def train_model(self):
        train_texts, train_labels = self._prepare_training_data()
        # Fit your model here.
        raise NotImplementedError

    def predict_one(self, claim_text, evidence_ids):
        evidence_texts = [
            self.evidence[eid]
            for eid in evidence_ids
            if eid in self.evidence
        ]
        combined_text = claim_text + " [SEP] " + " ".join(evidence_texts)

        # Return one of: SUPPORTS, REFUTES, NOT_ENOUGH_INFO, DISPUTED.
        raise NotImplementedError

    def evaluate(self):
        predictions = {}
        for claim_id, claim_data in self.dev_claims.items():
            claim_text = claim_data.get("claim_text", "")
            evidence_ids = claim_data.get("evidences", [])
            pred_label = self.predict_one(claim_text, evidence_ids)
            predictions[claim_id] = {
                "claim_label": pred_label,
                "evidences": list(evidence_ids),
            }

        predictions_path = OUTPUT_PATH / "predictions.json"
        with open(predictions_path, "w", encoding="utf-8") as f:
            json.dump(predictions, f, indent=2, ensure_ascii=False)
        print(f"Wrote eval predictions to {predictions_path}")

    def _load_retrieval_results(self, filename: str):
        output_file = OUTPUT_PATH / "retreived_data" / filename
        with open(output_file, "r") as f:
            retrieval_results = json.load(f)
        return retrieval_results
```

## Wiring The New Class Into The Pipeline

In `run.py`, replace the classifier import:

```python
from classifier import TfidfClaimClassifier
```

with your new class:

```python
from classifier import NewClaimClassifier
```

Then replace:

```python
classifier = TfidfClaimClassifier(retrieval_filename=file_name)
```

with:

```python
classifier = NewClaimClassifier(retrieval_filename=file_name)
```

Do not change the output format unless you also update `eval.py`.

## Running And Evaluating

From the repo root:

```bash
python run.py --retreiver-top-k 340 --reranker-top-k 50 --include_reranking True
```

Then evaluate:

```bash
python eval.py --predictions output/predictions.json --groundtruth output/dev-groundtruth.json --output_filename new-classifier-results.json
```

The evaluator reports:

- Evidence retrieval precision
- Evidence retrieval recall
- Evidence retrieval F-score
- Claim classification accuracy
- Harmonic mean of evidence F-score and classification accuracy

## Style Guidelines

Keep the same style as the current code:

- Use classes for model components.
- Store configuration and model state on `self`.
- Use helper methods with leading underscores for internal steps.
- Keep the public pipeline methods named `train_model`, `predict_one`, and `evaluate`.
- Use `load_data()` and `OUTPUT_PATH` instead of hard-coded absolute paths.
- Use `tqdm` when looping through many claims.
- Keep predictions JSON-compatible: normal Python strings, lists, and dictionaries.
- Preserve the current claim/evidence dictionary structure.

Avoid these changes unless there is a strong reason:

- Do not change the `data/*.json` schema.
- Do not change `output/predictions.json` schema.
- Do not rename the `retreiver` package.
- Do not make the classifier read `.env`.
- Do not write model caches into `data/`; use `output/` instead.

## Checklist For A Teammate

Before opening a PR or sharing results:

- The new classifier can be instantiated with `retrieval_filename=file_name`.
- `train_model()` runs without changing `run.py` pipeline order.
- `predict_one(claim_text, evidence_ids)` returns a label string.
- `evaluate()` writes `output/predictions.json`.
- `eval.py` runs successfully on the generated predictions.
- New dependencies are documented in this file or added to a future requirements file.
